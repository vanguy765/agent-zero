-- Function to take snapshot
CREATE OR REPLACE FUNCTION take_progress_snapshot(wo_id UUID)
RETURNS void AS $$
DECLARE
  total INTEGER;
  completed INTEGER;
  percentage DECIMAL(5,2);
BEGIN
  SELECT
    COUNT(*) FILTER (WHERE NOT not_applicable),
    COUNT(*) FILTER (WHERE checked AND NOT not_applicable)
  INTO total, completed
  FROM work_order_tasks
  WHERE work_order_id = wo_id;
  
  percentage := CASE WHEN total > 0 THEN (completed::DECIMAL / total * 100) ELSE 0 END;
  
  INSERT INTO work_order_progress_snapshots
    (work_order_id, total_tasks, completed_tasks, completion_percentage)
  VALUES
    (wo_id, total, completed, percentage);
END;
$$ LANGUAGE plpgsql;

CREATE VIEW work_order_progress AS
SELECT
  wo.id AS work_order_id,
  wo.unit_id,
  wo.trade,
  COUNT(*) FILTER (WHERE NOT t.not_applicable) AS total_tasks,
  COUNT(*) FILTER (WHERE t.checked AND NOT t.not_applicable) AS completed_tasks,
  ROUND(
    CASE
      WHEN COUNT(*) FILTER (WHERE NOT t.not_applicable) > 0
      THEN (COUNT(*) FILTER (WHERE t.checked AND NOT t.not_applicable)::DECIMAL /
            COUNT(*) FILTER (WHERE NOT t.not_applicable) * 100)
      ELSE 0
    END,
    2
  ) AS completion_percentage
FROM work_orders wo
LEFT JOIN work_order_tasks t ON t.work_order_id = wo.id
GROUP BY wo.id, wo.unit_id, wo.trade;

CREATE VIEW unit_progress AS
SELECT
  u.id AS unit_id,
  u.unit_number,
  u.project_id,
  COUNT(DISTINCT wo.id) AS total_work_orders,
  COUNT(DISTINCT wo.id) FILTER (WHERE wo.status = 'completed') AS completed_work_orders,
  ROUND(AVG(wop.completion_percentage), 2) AS avg_completion_percentage
FROM units u
LEFT JOIN work_orders wo ON wo.unit_id = u.id
LEFT JOIN work_order_progress wop ON wop.work_order_id = wo.id
GROUP BY u.id, u.unit_number, u.project_id;

# 📖 Measurements & Calculations System — Reference Guide

---

## Understanding Cost Types

You're right — not everything relates to square footage. The system handles **three distinct cost models**:

| Cost Model | Measured By | Examples |
|---|---|---|
| **Area-based** | $/SF or $/m² | Paint, flooring, drywall, ceiling tiles |
| **Linear-based** | $/LF | Baseboards, crown molding, caulking, countertop edges |
| **Per-item (count)** | $/each | Toilets, sinks, faucets, appliances, doors, light fixtures, outlets |

The `cost_per_sf` column in the views only applies to area-based costs. For a toilet at $350, that number stands alone — dividing it by room area would be meaningless.

---

## 🗄️ Table 1: `unit_room_measurements`

**What it is:** The physical reality of each room — dimensions you'd get from a tape measure or floor plan.

**One row per room per unit** (1:1 with `unit_rooms`).

| Column | Meaning | Example | How It's Used |
|---|---|---|---|
| `unit_room_id` | Links to which unit + room | Kitchen in Unit 201 | Joins everything together |
| `area_sf` | Floor area in square feet | 49.5 | Calculate flooring material needed, cost per SF comparisons |
| `area_m2` | Same area in metric | 4.60 | For metric-system suppliers or international reporting |
| `wall_length_total_lf` | Total wall perimeter | 28.2 LF | Starting point for baseboard, chair rail, wainscoting estimates |
| `wall_length_cabinets_lf` | Wall hidden by cabinets | 8.5 LF | Subtracted from paintable wall — you don't paint behind cabinets |
| `wall_length_paintable_lf` | Exposed wall length | 19.7 LF | Multiply by ceiling height = paintable wall area |
| `baseboard_length_lf` | Baseboard run | 19.7 LF | Order this many linear feet of baseboard material |
| `ceiling_height_ft` | Floor to ceiling | 8.0 | Multiplied by wall length = wall area for paint/drywall |
| `wall_area_sf` | Total wall surface | 157.6 SF | Paint coverage calculation (÷ 350 = gallons needed) |
| `ceiling_area_sf` | Ceiling surface | 49.5 SF | Ceiling paint, tiles, or drywall |
| `window_count` | Number of windows | 1 | Window replacement quotes, blind/curtain ordering |
| `door_count` | Number of doors | 1 | Door replacement quotes, subtract from baseboard length |
| `width_ft` / `depth_ft` | Room dimensions | 7.5 × 6.6 | Layout planning, furniture fit, appliance sizing |
| `width_mm` / `depth_mm` | Same in millimeters | 2255 × 2010 | Precision ordering for countertops, custom cabinetry |
| `source` | Where data came from | `pdf_extraction` | Audit trail — was this measured on-site or from drawings? |
| `extraction_confidence` | AI confidence score | 0.85 | Flag low-confidence rooms for manual verification |
| `extras` | JSONB overflow | `{"has_soffit": true}` | Anything unusual that doesn't fit standard columns |

### 💡 Practical Uses
- **"How much flooring do I need for the whole project?"** → `SELECT SUM(area_sf) * 1.10 FROM unit_room_measurements` (10% waste)
- **"Which rooms have the most paintable wall?"** → Sort by `wall_area_sf` descending
- **"Flag rooms where AI wasn't confident"** → Filter `extraction_confidence < 0.75`

## 🗄️ Table 2: `unit_room_calculations`

**What it is:** Your extensible cost ledger. Every supply estimate, material cost, labor hour, appliance price, and invoice allocation lives here.

**Multiple rows per room** — one row per metric. This is what makes it infinitely extensible.

| Column | Meaning | Example | How It's Used |
|---|---|---|---|
| `unit_room_id` | Which room | Bathroom in Unit 201 | Links to measurements |
| `metric_key` | Machine identifier | `appliance_cost` | Used in formulas and view pivots |
| `metric_label` | Human-readable name | "Appliance Cost" | What you see in NocoDB column headers |
| `category` | Grouping | `cost`, `supply`, `invoice`, `labor`, `measurement` | Filter in NocoDB: show me only costs, only supplies |
| `trade_key` | Which trade | `plumbing`, `painting`, `appliances` | Filter by trade, roll up trade totals |
| `value_numeric` | The number | 350.00 | The actual value — dollars, gallons, hours, count |
| `value_text` | Text/notes | "Kohler K-3817 toilet" | Product names, vendor notes, descriptions |
| `unit_of_measure` | Units | `$`, `gal`, `lf`, `ea`, `hrs` | Tells you what value_numeric means |
| `is_calculated` | Auto-computed? | `true` | If true, recalculate when inputs change |
| `is_editable` | User can change? | `true` | All rows are editable in NocoDB by default |
| `source` | Origin | `user_input`, `formula`, `invoice` | Track where numbers came from |
| `formula` | How it was computed | `wall_area_sf / 350` | Human-readable — helps you understand and verify |

### 💡 Examples of What Goes Here

**Area-based metrics:**
| metric_key | value_numeric | unit_of_measure | How calculated |
|---|---|---|---|
| `paint_wall_gal` | 0.45 | gal | wall_area_sf ÷ 350 (coverage rate) |
| `flooring_sf` | 54.5 | sf | area_sf × 1.10 (10% waste) |
| `flooring_cost` | 245.25 | $ | flooring_sf × cost_per_sf |

**Per-item metrics (NOT area-based):**
| metric_key | value_numeric | unit_of_measure | Notes |
|---|---|---|---|
| `appliance_cost` | 4,200 | $ | Fridge + stove + dishwasher — flat cost |
| `toilet_cost` | 350 | $ | Per unit, not per SF |
| `faucet_cost` | 185 | $ | Per item |
| `door_cost_per_unit` | 220 | $ | Per door × door_count |
| `light_fixture_cost` | 95 | ea/$ | Per fixture |

**Invoice proportioning:**
| metric_key | value_numeric | unit_of_measure | Notes |
|---|---|---|---|
| `sub_invoice_allocated` | 877.22 | $ | This room's share of a $50K painting contract |
| `sub_invoice_share_pct` | 27.6 | % | Based on wall area ratio |

**Labor:**
| metric_key | value_numeric | unit_of_measure | Notes |
|---|---|---|---|
| `labor_hours_est` | 4.5 | hrs | Estimated time for this room |
| `labor_rate_hr` | 45.00 | $/hr | Crew rate |
| `labor_cost` | 202.50 | $ | hours × rate |

### 💡 Practical Uses
- **"What's the total appliance budget?"** → Filter `metric_key = 'appliance_cost'`, sum `value_numeric`
- **"Show me everything the plumber needs"** → Filter `trade_key = 'plumbing'`
- **"What did the user manually enter vs what was calculated?"** → Filter by `source`
- **Add a new metric anytime** — just insert a row with a new `metric_key`. No migration needed.
