## Context

I'm launching **Bulwark AI** — my first open source project. It's a zero-dependency defensive coding standard (Guard-Trace-Verify) that makes AI-generated code more reliable. Think of it as a convention, not a library.

### Repository
- **GitHub**: `https://github.com/vanguy765/bulwark-ai` (currently private, needs to go public)
- **Local path**: `/a0/usr/workdir/bulwark-ai`
- **Tag**: `v0.2.0` (just tagged)
- **License**: Already has a LICENSE file

### What Makes It Interesting
1. **The Problem**: AI coding assistants generate code that looks correct but fails silently — wrong types, missing data, unverified side effects
2. **The Solution**: Three defensive layers (Guard inputs → Trace data shapes → Verify side effects) that any AI agent can learn from a single prompt
3. **Validation**: 6/6 live tests passing — a Claude agent learned the standard and produced compliant code with 100% self-correction rate

## 1. Executive Summary

### What We're Testing

An autonomous AI agent (Claude Opus 4.6 running inside Agent Zero) that:
1. **Receives** a coding task
2. **Loads** the BMAD Bulwark skill (`.bmad/skills/bulwark-defensive-coding.md`)
3. **Generates** code following the Guard-Trace-Verify (GTV) pattern
4. **Lints** its own output using `bulwark-lint --format json`
5. **Self-corrects** any violations detected
6. **Re-lints** to confirm compliance
