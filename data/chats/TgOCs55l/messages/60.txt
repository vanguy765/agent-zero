139:### Hono API — doc-parser routes (`/api/doc-parser/*`)
141:The Hono API (port 3000) wraps the doc-parser with Supabase integration.
142:All routes accept/return JSON. `document_id` must exist in `project_documents`.
148:| `POST` | `/api/doc-parser/pipeline` | **Full pipeline** — parse entire PDF, persist all pages + OCR to DB. Blocks until complete. |
149:| `POST` | `/api/doc-parser/pipeline/page` | **Single-page pipeline** — extract page N via `pdf-lib`, parse, upsert that page only. Body: `{ document_id, page_number, engine? }` |
150:| `POST` | `/api/doc-parser/pipeline/reprocess` | **Idempotent re-run** — same as `/pipeline` but safe to call multiple times (upsert). Adds `reprocessed: true` in response. |
156:| `POST` | `/api/doc-parser/parse` | Submit job, return `job_id` immediately (caller polls) |
157:| `GET`  | `/api/doc-parser/status/:jobId` | Lightweight poll — safe every 2-5s |
158:| `GET`  | `/api/doc-parser/results/:jobId` | Full results — fetch once after `completed` |
159:| `GET`  | `/api/doc-parser/health` | Health check including doc-parser reachability |
161:#### Read endpoints (DB queries — no doc-parser call)
165:| `GET`  | `/api/doc-parser/documents/:documentId/pages` | List all parsed pages. Query params: `?category=floorplan\|table\|site_photo\|diagram\|mixed\|text`, `?limit=`, `?offset=` |
166:| `GET`  | `/api/doc-parser/documents/:documentId/pages/:pageNumber` | Get one page by 1-indexed page number (storage_url, thumbnail, category, raw_text, metadata) |
167:| `GET`  | `/api/doc-parser/documents/:documentId/ocr` | List OCR/table results. Query params: `?page_number=N`, `?limit=`, `?offset=` |
168:| `GET`  | `/api/doc-parser/documents/:documentId/ocr/:pageNumber` | OCR results for a single page — returns page metadata + all OCR/table rows for that page |
174:| `GET`  | `/api/doc-parser/projects/:projectId/documents` | List all documents for a project — id, title, description, file_url, page_count, ocr_table_count, parsed status. Query: `?limit=`, `?offset=` |
175:| `GET`  | `/api/doc-parser/projects/:projectId/documents/:documentId` | Full document detail — metadata + page list with thumbnails + category breakdown + OCR table count |
177:> **Note:** `/pipeline/page` requires `pdf-lib`. Run `pnpm install` on Windows after any container-side `package.json` change.
