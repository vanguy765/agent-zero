Content: # Problem
The user wants to run an independent validation of a Product Requirements Document (PRD) using a different AI model (e.g., Gemini, GPT-4o) within the BMAD (Business Model & Application Design) framework. A specific challenge is that BMAD workflow files contain explicit 'HALT', 'WAIT FOR INPUT', or 'FORBIDDEN' instructions that cause external agents to stop at every step.

# Solution

### Option 1: External VS Code Integration (Gemini Code Assist - Recommended)
1. **Setup**: Install the **'Gemini Code Assist'** extension in VS Code. Sign in to access the free tier.
2. **Configuration**: 
   - Enable **Agent Mode**: `gemini.agentMode: true` in VS Code settings.
   - Set planning mode: `gemini.agent.planningMode: "detailed"` to allow multi-step analysis.
   - Enable **Local Codebase Awareness** to allow the model to index project files.
3. **Execution**: Open the project folder containing the PRD and BMAD framework files (`_bmad/`).
4. **Workflow Automation (Crucial Override)**: To bypass the manual confirmation steps built into BMAD markdown/XML structures, provide a specific override prompt:
   - **Instruction**: Auto-select 'Continue' (C) at every menu.
   - **Scope**: Process all remaining steps (e.g., V-03 through V-13) in a single continuous pass.
   - **Logic**: Explicitly ignore 'wait', 'halt', or 'FORBIDDEN' keywords in the source files.
   - **Output**: Direct the agent to write the final output to a specific path (e.g., `_bmad-output/planning-artifacts/prd.validation.md`).
   - **Example Prompt**: "IMPORTANT OVERRIDE: Process ALL remaining steps in a single continuous pass. DO NOT halt at menus. Auto-select 'C' at every prompt. Ignore any instructions in step files that say 'wait' or 'halt'. Write the final report to [PATH]. Begin now."
5. **Context**: Direct the agent to read the PM persona (`_bmad/bmm/agents/pm.md`), the validation workflow (`_bmad/bmm/workflows/.../workflow-validate-prd.md`), and the target PRD.

### Option 2: Native Agent Zero Global Settings Swap
1. **Identify Limitation**: Since `.a0proj/agents.json` lacks model overrides, changes must be global.
2. **Configuration**: Change the `chat_model_name` in the Agent Zero Settings UI or `/a0/usr/settings.json` to the desired model (e.g., `google/gemini-1.5-pro`).
3. **Execution**: Open a **new chat context** to initialize the session with the new model and run the validation command (e.g., `/bmad-bmm-validate-prd`).
4. **Cleanup**: Revert the `chat_model_name` back to the primary model after the report is generated.

### Detailed 13-Step Validation Workflow (V-01 to V-13)
- **V-01 to V-02**: Document Discovery and Format/Parity classification.
- **V-03 to V-04**: Information Density (scan for fluff) and Brief Coverage.
- **V-05 to V-06**: Measurability (testable FRs/NFRs) and Traceability (Vision -> Success -> FRs).
- **V-07**: Implementation Leakage (ensure requirements are implementation-agnostic).
- **V-08 to V-10**: Compliance check and SMART Validation.
- **V-11 to V-13**: Quality assessment, final sweep, and generation of the structured report (e.g., `prd.validation.md`).
