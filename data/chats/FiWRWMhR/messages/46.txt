## ADR-001: Project Name

**Date**: 2026-03 (v0.1.0)
**Status**: Accepted

### Context
The project needed a name that communicated defensive purpose, was memorable,
searchable, and available as a GitHub namespace. Two finalists emerged:
`assert-ai` and `bulwark-ai`.

### Decision
**`bulwark-ai`** was selected.

### Alternatives Considered
| Name | Pros | Cons |
|------|------|------|
| `assert-ai` | Familiar to developers, implies validation | Narrow semantic scope (assertions are just one pattern), poor SEO ("assert" is a language keyword with millions of results), suggests testing rather than defense |
| `bulwark-ai` | Strong defensive narrative, broader semantic scope, better searchability, evokes protection/fortification | Less immediately technical, requires brief explanation |

### Rationale
"Bulwark" better represents the mission across all three layers. An assertion
is a single check; a bulwark is a defensive structure. The name scales with
the project's ambition — from simple guards to a complete defensive membrane.
The word carries connotations of strength, protection, and deliberate

## ADR-019: BMAD Integration as Skill

**Date**: 2026-03 (v0.2.0-draft)
**Status**: Planned

### Context
The BMAD (Breakthrough Method for Agile AI-Driven Development) framework
uses agent roles and skills. Bulwark could integrate as a core requirement
or as an optional skill.

### Decision
Bulwark integrates as a **skill** in `.bmad/skills/`, not a core requirement.

### Rationale
- Skills are opt-in, matching Bulwark's additive philosophy
- The Developer agent and QA agent can both reference the Bulwark skill
- Skill format allows including the checklist and system prompt
- Doesn't require BMAD framework modifications
