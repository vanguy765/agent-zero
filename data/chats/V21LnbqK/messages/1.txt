## Project Overview

A **monorepo** for a construction industry platform. Core capability: voice-enabled data capture
for field workers, with AI extraction of structured data from audio and PDF documents.

**Stack:** Hono API · SolidJS frontends · Supabase (Postgres + Auth + Realtime) · Python doc-parser

## Architecture

```
voicelog-scaffold/
├── apps/
│   ├── api/              # Hono backend (The Brain) — port 3000
│   ├── web-dashboard/    # SolidJS admin — port 3001
│   ├── site-companion/   # SolidJS PWA (field workers) — port 3002
│   ├── material-admin/   # Material & taxonomy manager — port 3003
│   ├── work-orders/      # Work orders management — port 3004
│   └── tenant-portal/    # Tenant data portal — port 3005
├── doc-parser/           # Python FastAPI AI service — port 5000
├── packages/
│   ├── schemas/          # Zod schemas — THE source of truth
│   ├── db/               # Auto-generated Supabase TypeScript types
│   └── ui/               # Shared design system
└── supabase/             # Migrations, seed, config
```

**Development workflow:** Schema-first. Define data in `packages/schemas` → API route → UI.

## ⛔ Critical Agent Rules — doc-parser

**doc-parser is a standalone Docker container. It is NOT a local process inside the Agent Zero container.**

| Rule | Detail |
|------|--------|
| ❌ NEVER run `server.py` directly | It runs inside its own container, not here |
| ❌ NEVER install Python packages in `doc-parser/` | No pip, no venv, no requirements.txt installs from this container |
| ❌ NEVER look for or activate a venv in `doc-parser/` | There is no venv there — it lives inside the Docker image |
| ✅ If port 5000 is unreachable | `notify_user` type=warning and ask user to run `docker-compose up` in `doc-parser/` **on Windows** |
| ✅ To interact with doc-parser | Use HTTP via `http://doc-parser:5000` (shared `agent-zero-network`) or `http://localhost:5000` |
| ✅ To run CLI tools | Use `process_document.py` from this container pointing at the running API |

> **Why this matters:** This mistake has caused repeated harm — installing packages, trying to start the server locally, and wasting session time. When the health check fails, the ONLY correct action is to notify the user to start the container on Windows.

## Doc-Parser Service

**CRITICAL: Must run as Docker.** Do not run natively — requires system C++ deps (Tesseract, Poppler, OpenCV).

```bash
cd doc-parser
docker compose up -d --build
```

### Endpoints

| Method | Path               | Description                            |
|--------|--------------------|----------------------------------------|
| POST   | `/parse/docling`   | Docling-only engine (primary)          |
| POST   | `/parse/hybrid`    | Hybrid: Docling decompose + Claude extract |
| GET    | `/status/{job_id}` | Poll for progress (safe to call every 2-5s) |
| GET    | `/results/{job_id}`| Fetch full results once complete       |
| GET    | `/health`          | Dependency status check                |
| GET    | `/jobs`            | List all jobs                          |

### Hono API — doc-parser routes (`/api/doc-parser/*`)

The Hono API (port 3000) wraps the doc-parser with Supabase integration.
All routes accept/return JSON. `document_id` must exist in `project_documents`.

#### Pipeline endpoints (blocking — submit + poll + persist to DB)

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/doc-parser/pipeline` | **Full pipeline** — parse entire PDF, persist all pages + OCR to DB. Blocks until complete. |
| `POST` | `/api/doc-parser/pipeline/page` | **Single-page pipeline** — extract page N via `pdf-lib`, parse, upsert that page only. Body: `{ document_id, page_number, engine? }` |
| `POST` | `/api/doc-parser/pipeline/reprocess` | **Idempotent re-run** — same as `/pipeline` but safe to call multiple times (upsert). Adds `reprocessed: true` in response. |

#### Job control endpoints (async — caller manages polling)

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/doc-parser/parse` | Submit job, return `job_id` immediately (caller polls) |
| `GET`  | `/api/doc-parser/status/:jobId` | Lightweight poll — safe every 2-5s |
| `GET`  | `/api/doc-parser/results/:jobId` | Full results — fetch once after `completed` |
| `GET`  | `/api/doc-parser/health` | Health check including doc-parser reachability |

#### Read endpoints (DB queries — no doc-parser call)

| Method | Path | Description |
|--------|------|-------------|
| `GET`  | `/api/doc-parser/documents/:documentId/pages` | List all parsed pages. Query params: `?category=floorplan\|table\|site_photo\|diagram\|mixed\|text`, `?limit=`, `?offset=` |
| `GET`  | `/api/doc-parser/documents/:documentId/pages/:pageNumber` | Get one page by 1-indexed page number (storage_url, thumbnail, category, raw_text, metadata) |
| `GET`  | `/api/doc-parser/documents/:documentId/ocr` | List OCR/table results. Query params: `?page_number=N`, `?limit=`, `?offset=` |
| `GET`  | `/api/doc-parser/documents/:documentId/ocr/:pageNumber` | OCR results for a single page — returns page metadata + all OCR/table rows for that page |

#### Document manager endpoints (company/project scope)

| Method | Path | Description |
|--------|------|-------------|
| `GET`  | `/api/doc-parser/projects/:projectId/documents` | List all documents for a project — id, title, description, file_url, page_count, ocr_table_count, parsed status. Query: `?limit=`, `?offset=` |
| `GET`  | `/api/doc-parser/projects/:projectId/documents/:documentId` | Full document detail — metadata + page list with thumbnails + category breakdown + OCR table count |

> **Note:** `/pipeline/page` requires `pdf-lib`. Run `pnpm install` on Windows after any container-side `package.json` change.
#### Pipeline endpoint — quick reference
```bash
curl -X POST http://localhost:3000/api/doc-parser/pipeline \
  -H "Content-Type: application/json" \
  -d '{"document_id": "<uuid>", "engine": "hybrid"}'
# Returns: { document_id, job_id, pages_persisted, ocr_tables_persisted, summary }
```
> Full docs: `.agents/workflows/parse-document.md`

### Pipeline (Docling engine)
1. **Docling** — extracts text, tables, images from PDF
2. **Classification** — Tesseract + color variance separates floorplans / site photos / diagrams
3. **LLM analysis** — Claude 3.5 Sonnet (via OpenRouter) extracts rooms, doors, windows, areas
4. **Cross-page synthesis** — second LLM call consolidates multi-page results
5. **Confidence scoring** — quality score attached to result

### LLM Provider
Default: `anthropic/claude-3.5-sonnet` via OpenRouter (LiteLLM under the hood).
Override with `DEFAULT_LLM_PROVIDER` env var.

```powershell

cd doc-parser && docker compose up -d
```

powershell -File scripts/kill-dev-servers.ps1

## Supabase Troubleshooting

- **Realtime not updating**: Check RLS policies and that the table is in `supabase_realtime` publication.
- **`supabase start` parse errors**: Comment out `deno_version` / `health_timeout` in `supabase/config.toml`.
- **`major_version`**: Must be `17`. Do not downgrade.
