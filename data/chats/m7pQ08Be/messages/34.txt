1103:## 7. Common Mistakes
1109:### Mistake 1: Silent Coercion with `data = data or {}`
1114:# ❌ WRONG — silently masks missing data
1126:# ✅ RIGHT — fails loudly if config is missing
1150:### Mistake 2: Catching Exceptions and Ignoring Them
1155:# ❌ WRONG — swallows the error, continues with bad state
1169:# ✅ RIGHT — catches specific errors, fails loudly for unexpected ones
1201:### Mistake 3: Logging Actual Data Values in Traces
1206:# ❌ WRONG — logs actual data content (security risk!)
1211:# This might output:
1212:# TRACE:after_fetch | data={'user': 'alice', 'ssn': '123-45-6789', 'password': 'hunter2'}
1220:# ✅ RIGHT — logs shape only, never content
1226:# This outputs:
1227:# TRACE:after_fetch | dict, keys=['password', 'ssn', 'user'], len=3
1234:### Mistake 4: Verifying the Status Code but Not the Response Body
1239:# ❌ WRONG — trusts the status code blindly
1251:# ✅ RIGHT — verifies the response body contains the expected result
1284:### Mistake 5: Using Bare `assert` in Python Production Code
1289:# ❌ WRONG — assert is stripped by python -O
1302:# ✅ RIGHT — explicit if/raise is never stripped
1327:# ✅ ALSO RIGHT — helpers use if/raise under the hood
