##### Recovery MUST Rules

- **R1: Recovery MUST be optional.** Omitting `recovery_command` MUST produce identical behavior to the pre-recovery Availability Guard. No existing code breaks.
- **R2: Recovery MUST be bounded.** The guard MUST NOT retry indefinitely. `max_retries` MUST be enforced.
- **R3: Recovery MUST be logged.** Every recovery attempt MUST produce a diagnostic trace including: attempt number, max attempts, command executed, command exit code, and elapsed time.
- **R4: Recovery commands MUST be explicit.** The guard MUST NOT infer or construct recovery commands. The caller provides the exact command.
- **R5: Recovery MUST NOT escalate privileges.** The command runs with the current process permissions. No `sudo` injection, no privilege elevation.
- **R6: Recovery failure MUST include full context.** The final failure diagnostic MUST include: dependency name, all attempts made, total time elapsed, last command exit code, and the original availability check result.

##### Pattern

```python
# Detection only (no recovery) — existing behavior, unchanged
guard_available(
    check=lambda: requests.head(url, timeout=5).status_code < 500,
    function_name="app_startup",
    dependency="payment-service",
)

# Detection with recovery — new capability
guard_available(
    check=lambda: requests.head(url, timeout=5).status_code < 500,
    function_name="app_startup",
    dependency="payment-service",
    recovery_command="docker compose up -d payment-service",
    max_retries=3,
    retry_delay=5.0,
    backoff_multiplier=2.0,
)

# Recovery with a multi-step script
guard_available(
    check=lambda: requests.head(url, timeout=5).status_code < 500,
    function_name="pipeline_start",
    dependency="full-stack",
    recovery_command="/scripts/reset-and-start.sh",
    max_retries=2,
    retry_delay=10.0,
)
```

##### Diagnostic Format

Recovery attempts produce structured log lines:

```
GUARD:AVAILABLE:app_startup | payment-service — unavailable, attempting recovery
GUARD:RECOVERY:app_startup | payment-service — executing "docker compose up -d payment-service" (attempt 1/3)
GUARD:RECOVERY:app_startup | payment-service — command exited with code 0, waiting 5.0s
GUARD:AVAILABLE:app_startup | payment-service — OK (recovered after 1 attempt, 8.2s elapsed)
```

Recovery failure:

```
GUARD:AVAILABLE:app_startup | payment-service — unavailable, attempting recovery
GUARD:RECOVERY:app_startup | payment-service — executing "docker compose up -d payment-service" (attempt 1/3)
GUARD:RECOVERY:app_startup | payment-service — command exited with code 0, waiting 5.0s
GUARD:AVAILABLE:app_startup | payment-service — still unavailable after attempt 1
GUARD:RECOVERY:app_startup | payment-service — executing "docker compose up -d payment-service" (attempt 2/3)
