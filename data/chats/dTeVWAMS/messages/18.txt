(shape mismatches, unverified side effects) occur in hand-written code too.
AI code just has them more frequently.

### OQ-6: Recovery Command Security
**Question**: How to handle security implications of shell command execution
in Recovery?

**Context**: `guard_available()` with `recovery_command` executes arbitrary
shell commands. In production environments, this could be a security concern:
- Should recovery commands be restricted to a whitelist?
- Should they run in a sandboxed environment?
- Is shell execution sufficient or should we support callable functions?

**Leaning**: Document the security implications clearly. Recommend that
recovery commands be defined in configuration, not in code. Support
callable functions as an alternative to shell commands in a future version.

### OQ-7: Versioning Strategy
**Question**: How to version the specification vs. implementations?

**Context**: The spec and implementations may evolve at different rates.
- Should SPEC.md have its own version number?
- Should reference implementations track the spec version?
- What constitutes a breaking change in a convention-based standard?

**Leaning**: Semantic versioning for the spec. Reference implementations
track the spec version they implement. Breaking change = removing or
changing the behavior of an existing function.
